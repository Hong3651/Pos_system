# POS SaaS 시스템 - 개발 계획 (Plan)

기술 스택과 요구사항은 spec.md 참조.

## 프로젝트 구조
```
pos_system/
├── backend/                 # Django + DRF API 서버
│   ├── config/              # settings, urls, wsgi
│   └── apps/
│       ├── accounts/        # 사용자, 인증, SMS
│       ├── stores/          # 가게, 직원, 설정
│       ├── products/        # 상품, 카테고리
│       └── sales/           # 판매, 환불, 현금영수증, 일마감, 통계
├── frontend/                # Next.js 프론트엔드
│   └── src/
│       ├── app/             # App Router 페이지
│       ├── components/      # 공통 컴포넌트
│       ├── hooks/           # 커스텀 훅
│       ├── lib/             # API 클라이언트, 유틸
│       └── types/           # TypeScript 타입
├── .github/workflows/       # GitHub Actions (CI/CD)
├── spec.md                  # 요구사항 명세서
├── plan.md                  # 개발 계획 (이 파일)
└── README.md
```

## 테스트 전략
- **매 Phase 완료 시** 해당 기능의 테스트 작성
- 백엔드: Django TestCase (모델 단위 + API 테스트)
- 프론트엔드: Jest + React Testing Library (컴포넌트 + 페이지)
- 금액/VAT/도매가/재고 로직은 반드시 테스트 포함

- [ ] Django 프로젝트 생성 (backend/)

- [ ] PostgreSQL 연결 설정 (환경변수 기반)
- [ ] DRF 설정 (JWT, 페이지네이션, 예외 처리)
- [ ] Sentry 연동 (백엔드)
- [ ] Next.js 프로젝트 생성 (frontend/)
- [ ] Tailwind CSS 설정
- [ ] API 클라이언트 설정 (axios + 토큰 자동 첨부)
- [ ] Sentry 연동 (프론트엔드)
- [ ] 환경변수 관리 (.env)

### Phase 2: 인증 시스템

- [ ] 비밀번호 정책 검증 (8자+, 2종류+, 아이디 겹침 방지)
- [ ] SMS 인증코드 발송/검증 API (가입 시 1회)
- [ ] JWT 발급/갱신/로그아웃 API
- [ ] 로그인 보안 (5회 실패 시 30분 잠금)
- [ ] 프론트: 회원가입 페이지 (이름, 아이디, 비밀번호, 전화번호 인증)
- [ ] 프론트: 로그인 페이지 (아이디 + 비밀번호, 잠금 안내)
- [ ] 프론트: 인증 상태 관리 (토큰 저장/갱신)
- [ ] 인증 미들웨어 (비로그인 시 로그인 페이지로)
- [ ] **테스트**: 회원가입, 로그인, 비밀번호 정책, 잠금 로직

### Phase 5: 판매/체크아웃 (핵심)
- [ ] Sale 모델 (store, staff, 금액, VAT, 결제수단, 카드승인정보, transaction_id, status)

- [ ] 프론트: 체크아웃 페이지 (2단 레이아웃)

  - 결제 모달 (현금/카드/혼합, 거스름돈)

- [ ] 결제 완료 → 영수증/거래명세서 발행 선택
- [ ] **테스트**: 판매 처리, VAT 계산, 도매가 적용, 재고 차감, 환불(전체/부분), 이중결제 방지

## 구현 우선순위
1. **Phase 1~2** (인프라 + 인증) — 다른 모든 것의 기반
2. **Phase 3** (가게 + 직원) — 멀티테넌트 핵심
3. **Phase 4~5** (상품 + 판매) — 실제 비즈니스 로직
4. **Phase 6~7** (이력 + 통계 + 일마감) — 데이터 활용
5. **Phase 8** (거래처 + 거래명세서) — B2B 기능
6. **Phase 9~10** (활동로그 + 설정 + PWA + 배포) — 런칭 준비

핵심 설계 원칙은 CLAUDE.md 참조.

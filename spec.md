# 철물점 POS 시스템 - 요구사항 명세서 (Specification)

## 1. 프로젝트 개요
- **프로젝트명**: 철물점 POS 시스템
- **목적**: 철물점 운영에 최적화된 웹 기반 판매/결제 시스템
- **사용자**: 철물점 사장님 (비개발자)
- **플랫폼**: 웹 애플리케이션 (PC, 태블릿, 모바일 브라우저)

## 2. 기술 스택
| 구분 | 기술 | 비고 |
|------|------|------|
| Backend | Python 3 + Flask | 경량 웹 프레임워크 |
| Database | SQLite3 | 내장 DB, 설치 불필요 |
| Frontend | HTML/CSS/JS + Bootstrap 5 | 반응형, 터치 친화적 |
| 차트 | Chart.js | 매출 통계 시각화 |

## 3. 핵심 기능 요구사항

### 3.1 판매/결제 (Checkout)
- 상품 검색 (이름, 바코드)
- USB 바코드 스캐너 지원
- 장바구니 관리 (추가, 수량변경, 삭제)
- 도매가 자동 적용 (수량 기준)
- 결제 처리 (현금, 카드, 혼합)
- 거스름돈 자동 계산
- 영수증 인쇄 (브라우저 인쇄)
- 키보드 단축키 지원

### 3.2 상품 관리
- 상품 등록/수정/삭제 (소프트 삭제)
- 카테고리 분류
- 단위 유형: 개, 박스, 통, 포, m, kg, 롤, 장, 세트
- 도매가 설정 (기준 수량 + 도매 단가)
- 재고 수량 표시

### 3.3 매출 통계
- 일별/월별 매출 현황 (차트)
- 인기 상품 분석 (판매량/매출액 기준)
- 카테고리별 매출 비율
- 현금/카드 결제 비율

### 3.4 대시보드
- 오늘 매출 합계
- 오늘 거래 건수
- 현금/카드 비율
- 재고 부족 알림

## 4. 결제 수단
- **현금**: 받은 금액 입력 → 거스름돈 자동 계산
- **카드**: 카드 결제 금액 기록 (실제 카드 단말기 연동 X)
- **혼합**: 현금 + 카드 분할 결제

## 5. 철물점 특화 요구사항
- 다양한 판매 단위 지원 (개수, 길이(m), 무게(kg))
- 소수점 수량 지원 (케이블 3.5m 등)
- 도매/대량 구매 할인 자동 적용
- 철물점 기본 카테고리 제공

## 6. 비기능 요구사항
- 오프라인 동작 (로컬 서버, 외부 CDN 의존 없음)
- `python pos.py` 한 줄로 실행
- 데이터 자동 백업 지원
- 한국어 UI
